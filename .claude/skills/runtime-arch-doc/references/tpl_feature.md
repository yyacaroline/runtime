**写作要点：**

1. **必须包含架构图**：使用 mermaid（graph TB/flowchart TD/sequenceDiagram）绘制分层架构和流程
2. **表格化接口和模块职责**：便于快速查阅
3. **标注代码位置**：文件路径标注如 `src/runtime/feature/...`
4. **包含实际代码示例**：关键代码片段用代码块展示
5. **设计决策说明**：说明"为什么这样设计"，而非仅描述"是什么"
6. 严格按照代码逻辑，禁止捏造
7. 更新文档时，要尊重原文档内容。当发现与原文档内容有冲突时，输出原文和要修改的内容后提示用户，确认后再进行修改

**特性设计文档格式：**

```markdown
# xx [特性名称] 特性

## 1. 特性概述
- **特性介绍**：简述特性的功能、价值
- **问题背景**：为什么需要这个特性？解决什么痛点？
- **设计目标**：列出核心设计目标（性能、可用性、兼容性等）

## 2. 使用场景与对外接口

### 2.1 使用场景
- 场景一：典型使用流程
- 场景二：特殊配置场景

### 2.2 对外接口
- 使用表格列出接口、Option、API：
  | 接口 | 文件位置 | 说明 |
- 包含示例代码

## 3. 架构总览
- 整体设计思路（跨哪些模块）
- 架构分层图（使用 mermaid graph TB 或 flowchart）
- 核心模块交互图（使用 mermaid sequenceDiagram 或 flowchart）

## 4. 详细设计/具体实现（可多个并列）

### 4.1 核心流程
- 使用 mermaid flowchart 或 sequenceDiagram
- 包含关键代码片段（标注文件位置）

### 4.2 核心机制详解
- 逐个介绍核心机制、策略
- 包含详细流程图（使用 mermaid flowchart TD）
- 关键代码片段（标注文件位置）

### 4.3 模块职责划分（如有）
- 使用表格列出各模块职责：
  | 模块 | 职责 | 位置 |

### 4.4 核心数据结构（如有）
- 使用 mermaid classDiagram/或者表格
- 说明关键类、成员变量、方法

## 5. 关键设计思想/设计思考/总结（如有）
- 核心设计原则（如性能优化、兼容性保障）
- 设计权衡（为什么这样设计，而非其他方案）

## 6.关键文件索引
- 使用表格列出相关文件位置：
  | 模块 | 文件路径 | 核心内容 |

## 7. 性能优化策略（如有）
- 该模块的性能优化手段

## 8. 兼容性与扩展性（如有）
- 模块边界稳定性
- 接口演化策略
- 扩展点说明

## 9. 典型问题与排查（如有）
- 关键日志点
- 调试开关
- 问题定位方法
```
